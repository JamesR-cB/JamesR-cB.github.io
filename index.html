Build environment prerequisites:

You will need the following dependencies installed in order to work with most, if not all projects.

  1. Visual Studio 2019 Community Edition for Windows
  2. SFML (ask James for the dlls, this is temporary)
  3. C++ build environment for Desktop installed in VS2019

Individual projects will list of dependencies that only they have.

Style Guide

There's a few standards to how I write my code. Most of it is for visual flair, though there are some code designs that will affect what you write. Every file starts with a header of comments, describing the file, what project it's for, and giving some documentation for how things look and feel. Comments are marked with the username of who leaves them, and some projects will also label what version of the project they are from. For example, Crowbar Engine has comments labeled for Crowbar 2018, Crowbar 2020, and Synchronic.

Namespaces are not used anywhere in the code. Namespaces can make the code simpler to read, but I don't like the way it looks and I don't like the ambiguity it can give. I realize this is not a standard thing to do, but I do the "extra work" for my own sake - it's helpful to glance at and see the full line. Yes, I'm strange.
